## 1. 시스템 접속

### 1.1 접속 URL
| 환경 | URL |
|------|-----|
| 관리자 포털 | `http://localhost:5173/admin/login` (개발) |
| 관리자 포털 | `http://{서버주소}/admin/login` (운영) |
| API 문서 | `http://localhost:8000/docs` |

### 1.2 관리자 로그인

1. 브라우저에서 **관리자 로그인** 페이지에 접속합니다
2. 아래 정보를 입력합니다:
   - **이메일**: `admin@baikal.ai`
   - **비밀번호**: `admin1234`
3. **로그인** 버튼을 클릭합니다
4. 로그인 성공 시 자동으로 **대시보드**로 이동합니다

> ⚠️ **보안 주의**: 운영 환경에서는 반드시 기본 비밀번호를 변경하세요.

### 1.3 로그아웃
- 좌측 사이드바 하단의 **로그아웃** 버튼을 클릭합니다
- 로그아웃 후 로그인 페이지로 이동합니다

## 2. 대시보드

로그인 직후 표시되는 **종합 현황판**입니다. 시스템의 전반적인 상태를 한눈에 파악할 수 있습니다.

### 2.1 통계 카드 (6개)

| 카드 | 설명 | 예시 |
|------|------|------|
| 🔹 **전체 신청** | 접수된 전체 신청서 수 | 14건 |
| 🔹 **진행중 사업** | 현재 접수 중인 사업 수 | 3개 |
| 🔹 **선정률** | 전체 대비 선정 비율 | 21.4% |
| 🔹 **선정 건수** | 선정(approved) 상태 신청 수 | 3건 |
| 🔹 **등록 사용자** | 전체 등록 사용자 수 | 8명 |
| 🔹 **총 예산** | 전체 사업 예산 합계 | 4.5억 |

### 2.2 차트

#### 상태별 신청 현황 (막대 차트)
- X축: 신청 상태 (접수, 검토중, 보완요청, 선정, 탈락, 정산완료)
- Y축: 건수
- 각 상태별 신청 건수를 시각적으로 비교

#### 상태 비율 (도넛 차트)
- 전체 신청서의 상태 분포를 파이/도넛 차트로 표시
- 각 상태별 비율 확인 가능

### 2.3 사업별 통계 테이블

| 컬럼 | 설명 |
|------|------|
| 사업명 | 지원사업 제목 |
| 상태 | 진행중 / 마감 |
| 신청 수 | 해당 사업 접수 건수 |
| 선정 수 | 해당 사업 선정 건수 |
| 예산 | 사업 총 예산 |

### 2.4 최근 활동 피드
- 최근 워크플로우 변경 이력 표시
- 누가(actor) + 어떤 작업(action) + 언제(timestamp) 를 시간순 나열
- 예: "홍길동 → 접수 | 2분 전"

### 2.5 최근 신청 목록
- 최근 접수된 신청서 테이블
- 접수번호, 신청자명, 사업명, 상태, 신청일 표시
- 클릭하여 상세 페이지로 이동

## 3. 지원사업 관리

좌측 메뉴 **사업 관리**를 클릭하여 접근합니다.

### 3.1 사업 목록 화면
- 등록된 전체 사업을 카드 형태로 표시
- 각 카드에 사업명, 상태 뱃지, 접수 기간, 예산 표시

### 3.2 신규 사업 등록

1. 우측 상단 **+ 새 사업** 버튼을 클릭합니다
2. 모달(팝업)에서 아래 정보를 입력합니다:

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| 사업명 | ✅ | 지원사업 이름 | 2026 AI 스타트업 혁신 지원사업 |
| 설명 | - | 사업 상세 설명 | AI·데이터 기반 혁신 기술... |
| 사업 시작일 | - | 사업 수행 시작 | 2026-04-01 |
| 사업 종료일 | - | 사업 수행 종료 | 2026-12-31 |
| 접수 시작일 | - | 신청 접수 시작 | 2026-03-01 |
| 접수 마감일 | - | 신청 접수 마감 | 2026-03-31 |
| 예산 | - | 총 지원 예산 (원) | 100000000 |
| 상태 | ✅ | 사업 상태 | 초안 / 진행중 / 마감 |

3. **저장** 버튼을 클릭합니다

### 3.3 사업 상태 설명

| 상태 | 뱃지 색상 | 설명 |
|------|-----------|------|
| **초안 (draft)** | 회색 | 준비 중, 사용자에게 노출되나 신청 불가 |
| **진행중 (active)** | 파란색 | 접수 진행 중, 사용자 신청 가능 |
| **마감 (closed)** | 빨간색 | 접수 종료 |

### 3.4 사업 수정
- 사업 카드의 **✏️ 수정(연필) 아이콘**을 클릭합니다
- 모달에서 정보를 수정하고 저장합니다

### 3.5 사업 삭제
- 사업 카드의 **🗑️ 삭제(휴지통) 아이콘**을 클릭합니다
- 확인 대화상자에서 **삭제**를 선택합니다

> ⚠️ **주의**: 사업 삭제 시 연결된 폼, 신청서가 모두 삭제됩니다. 신중하게 진행하세요.

### 3.6 폼 빌더 이동
- 사업 카드의 **⚙️ 설정(톱니바퀴) 아이콘**을 클릭하면 해당 사업의 **신청서 양식 설계** 화면으로 이동합니다

## 4. 신청서 양식 설계 (폼 빌더)

사업 관리 화면에서 **⚙️ 아이콘** 클릭 또는 URL `/admin/programs/{id}/form`으로 접근합니다.

### 4.1 폼 생성
- 해당 사업에 아직 폼이 없는 경우 **"신청서 양식 만들기"** 버튼이 표시됩니다
- 클릭하면 `"{사업명} 신청서"` 제목으로 폼이 자동 생성됩니다

### 4.2 필드 추가

1. 좌측의 **필드 추가** 패널에서 정보를 입력합니다:

| 설정 항목 | 설명 |
|-----------|------|
| **필드 유형** | 입력 방식 선택 (아래 참조) |
| **라벨** | 필드 제목 (사용자에게 보이는 이름) |
| **설명** | 필드 보조 설명 (선택) |
| **필수 여부** | 체크 시 반드시 입력해야 제출 가능 |
| **선택지** | select/checkbox 유형일 때 옵션 (줄바꿈 구분) |

2. **필드 추가** 버튼을 클릭합니다

### 4.3 지원하는 필드 유형 (7종)

| 유형 | 아이콘 | 설명 | 사용 예시 |
|------|--------|------|-----------|
| **텍스트 입력** | 📝 | 자유 텍스트 입력 | 기업명, 사업 개요 |
| **숫자 입력** | 🔢 | 숫자 전용 입력 | 매출액, 직원 수 |
| **선택박스** | ▼ | 드롭다운 단일 선택 | 업종 분류, 기업 규모 |
| **체크박스** | ☑️ | 다중 선택 가능 | 보유 인증, 관심 분야 |
| **날짜 입력** | 📅 | 달력 선택 | 설립일, 예정 시작일 |
| **파일 업로드** | 📎 | 첨부파일 | 사업계획서, 재무제표 |
| **서약서 동의** | ✅ | 동의 체크 | 개인정보 수집 동의 |

### 4.4 필드 관리
- **순서 변경**: 각 필드의 순서 번호로 표시 순서 확인
- **필드 삭제**: 각 필드 우측의 **🗑️ 삭제** 버튼 클릭

### 4.5 폼 설계 예시 (AI 스타트업 지원사업)

```
1. [텍스트] 기업/단체명 * (필수)
2. [텍스트] 대표자명 *
3. [숫자]  설립년도 *
4. [선택]  기업 규모 (스타트업/중소기업/중견기업) *
5. [텍스트] 주력 AI 기술 분야 *
6. [텍스트] 사업 개요 (500자 이내) *
7. [숫자]  신청 금액 (만원) *
8. [숫자]  최근 매출액 (만원)
9. [숫자]  R&D 인력 수
10. [파일]  사업계획서 *
11. [파일]  재무제표
12. [동의]  개인정보 수집·이용 동의 *
```

## 5. 신청 현황 관리

좌측 메뉴 **신청 현황**을 클릭하여 접근합니다.

### 5.1 신청 목록 화면

#### 상태 퀵 필터
- 화면 상단에 상태별 버튼이 표시됩니다
- 각 버튼에 해당 상태의 건수가 표시됩니다
- 클릭하면 해당 상태의 신청서만 필터링됩니다

```
[전체 14] [접수 3] [검토중 2] [보완요청 1] [선정 3] [탈락 2] [정산완료 3]
```

#### 사업별 필터
- **전체 사업** 드롭다운에서 특정 사업만 선택하여 필터링 가능

#### 목록 테이블

| 컬럼 | 설명 |
|------|------|
| 접수번호 | 고유 접수번호 (APP-XXXX-XXXXXXXX-XXXX) |
| 신청일 | 제출 일시 |
| 상태 | 현재 처리 상태 (컬러 뱃지) |
| AI 점수 | AI 자동 심사 점수 (색상으로 구분) |
| 관리 | 상세보기 / PDF 다운로드 버튼 |

#### AI 점수 색상 구분
| 점수 | 색상 | 의미 |
|------|------|------|
| 75점 이상 | 🟢 초록 | 선정 추천 |
| 50~74점 | 🟡 노랑 | 추가 검토 |
| 49점 이하 | 🔴 빨강 | 반려 권고 |

### 5.2 PDF 다운로드
- 목록의 **⬇ 다운로드** 아이콘을 클릭하면 해당 신청서의 PDF가 다운로드됩니다
- 파일명: `{접수번호}.pdf`

## 6. 신청서 상세 심사

신청 목록에서 **👁 상세** 버튼을 클릭하여 접근합니다.

### 6.1 화면 구성 (3컬럼)

```
┌───────────────────────────────────┬──────────────────┐
│         메인 콘텐츠 (2/3)          │  AI 분석 (1/3)    │
│                                   │                  │
│  📋 접수 정보                      │  🔵 AI 종합점수   │
│  📝 신청 내용 (답변)                │     (원형 그래프)  │
│  📎 첨부파일                       │  📊 카테고리 점수  │
│  🔄 상태 변경                      │  💡 AI 인사이트   │
│  📜 처리 이력                      │  ⚠️ 리스크 평가   │
└───────────────────────────────────┴──────────────────┘
```

### 6.2 신청 내용 확인
- 신청자가 작성한 모든 답변을 필드별로 표시합니다
- 첨부파일 목록과 다운로드 링크를 제공합니다

### 6.3 AI 심사 결과 확인

#### AI 종합 점수
- 원형 프로그레스 그래프로 종합 점수 (0~100) 표시
- 추천 결과: **선정 추천** / **추가 검토 필요** / **반려 권고**

#### 카테고리별 점수 (5개)
| 카테고리 | 바 차트 | 설명 |
|----------|---------|------|
| 서류 완성도 | ████████░░ 80 | 제출 서류 품질 |
| 사업 타당성 | ███████░░░ 72 | 계획 실현 가능성 |
| 기술 혁신성 | █████████░ 91 | 기술 차별화 |
| 재무 건전성 | ██████░░░░ 65 | 재무 상태 |
| 자격 적합성 | ████████░░ 85 | 자격 요건 충족 |

#### AI 인사이트
- AI가 분석한 주요 의견 3건 표시
- 예: "제출 서류가 충실하며 사업 계획이 구체적입니다"

#### 리스크 평가
| 리스크 | 수준 | 설명 |
|--------|------|------|
| 서류 누락 위험 | 낮음 🟢 | 서류 완성도가 높음 |
| 자격 미달 위험 | 낮음 🟢 | 자격 요건 충족 |
| 중복 수혜 의심 | 주의 🟡 | 확인 필요 |

### 6.4 상태 변경

1. **상태 변경** 패널에서 새로운 상태를 선택합니다:

| 변경 가능 상태 | 설명 |
|----------------|------|
| 검토중 (under_review) | 심사 시작 |
| 보완요청 (revision_requested) | 서류 보완 필요 |
| 선정 (approved) | 심사 통과 |
| 탈락 (rejected) | 심사 미통과 |
| 정산완료 (completed) | 사업 정산 완료 |

2. **사유/메모**를 입력합니다 (선택)
   - 예: "사업계획서 내 예산 항목 보완 필요"
3. **상태 변경** 버튼을 클릭합니다

> 📝 모든 상태 변경은 **워크플로우 로그**에 자동 기록됩니다 (변경자, 이전/이후 상태, 사유, 시간)

### 6.5 처리 이력 (워크플로우 로그)
- 해당 신청서의 모든 상태 변경 이력을 **타임라인** 형식으로 표시합니다
- 각 이력에 포함되는 정보:
  - 변경 일시
  - 이전 상태 → 새 상태
  - 변경자
  - 코멘트/사유

```
[2026-02-28 12:00] 제출 → submitted
  └ 홍길동 — "신청서 제출"

[2026-02-28 14:30] submitted → under_review
  └ admin@baikal.ai — "1차 서류 심사 시작"

[2026-02-28 16:00] under_review → approved
  └ admin@baikal.ai — "AI 심사 통과, 전문가 검토 완료"
```

### 6.6 PDF 다운로드
- 상세 화면 상단의 **📄 PDF** 버튼을 클릭하면 해당 신청서의 PDF를 다운로드할 수 있습니다

## 7. 자주 묻는 질문

### Q1. 사업 상태를 변경하면 이미 접수된 신청서에 영향이 있나요?
A. 사업 상태를 **마감(closed)**으로 변경해도 기존 접수된 신청서는 그대로 유지됩니다. 다만 새로운 신청은 불가능해집니다.

### Q2. 삭제한 사업을 복구할 수 있나요?
A. 현재 버전에서는 삭제된 사업의 복구 기능을 지원하지 않습니다. 삭제 전 반드시 확인하시기 바랍니다.

### Q3. 여러 관리자가 동시에 같은 신청서를 처리해도 되나요?
A. 가능합니다. 다만 상태 변경은 마지막 변경이 적용되므로, 동시 작업 시 처리 이력을 확인하고 진행하시기 바랍니다.

### Q4. AI 점수는 어떻게 산정되나요?
A. 현재 MVP 버전에서는 시뮬레이션 점수가 표시됩니다. 향후 v1.2 업데이트에서 실제 AI 모델이 연동되어 서류 분석 기반의 정밀 점수가 제공됩니다.

### Q5. 파일 첨부 용량 제한이 있나요?
A. 기본 설정상 개별 파일 10MB, 전체 100MB까지 업로드 가능합니다. 운영 환경에서 설정 변경이 가능합니다.

### Q6. 비밀번호를 변경하려면 어떻게 하나요?
A. 현재 MVP 버전에서는 비밀번호 변경 기능이 제공되지 않습니다. DB 직접 접근 또는 향후 업데이트를 통해 지원 예정입니다.

### Q7. 접수번호 체계는 어떻게 되나요?
A. `APP-{사업ID(4자리)}-{접수일(YYYYMMDD)}-{순번(4자리)}` 형식입니다.  
예: `APP-0001-20260228-0003` = 1번 사업, 2026년 2월 28일, 3번째 접수

```
1. 사업 등록
   └→ 사업 기본 정보 입력 (제목, 기간, 예산)
   └→ 상태: "초안"

2. 신청서 양식 설계
   └→ 폼 빌더에서 필드 추가 (텍스트, 파일, 선택 등)
   └→ 필드 순서 및 필수 여부 설정

3. 사업 공개
   └→ 사업 상태를 "진행중"으로 변경
   └→ 사용자 포털에서 조회 및 신청 가능

4. 신청 접수 관리
   └→ 대시보드에서 접수 현황 모니터링
   └→ 신청 현황에서 상세 확인

5. 심사 진행
   └→ AI 점수 참고하여 서류 검토
   └→ 상태 변경: 검토중 → 선정/탈락/보완요청

6. 결과 관리
   └→ 선정 결과 확정
   └→ 정산 완료 처리

7. 사업 마감
   └→ 사업 상태를 "마감"으로 변경
```
