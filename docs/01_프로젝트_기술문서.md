## 1. 프로젝트 개요

### 1.1 프로젝트명
**BAIKAL Grant AI** — AI 기반 보조금·지원사업 접수 및 관리 시스템

### 1.2 프로젝트 목적
정부·지자체·공공기관의 보조금 및 지원사업 신청-접수-심사-선정-정산 전 과정을 디지털화하고, AI 기술을 활용하여 심사 효율성과 투명성을 극대화하는 통합 관리 플랫폼 구축

### 1.3 핵심 가치
| 항목 | 설명 |
|------|------|
| **AI 자동 심사** | 서류 완성도, 사업 타당성, 기술 혁신성 등 5개 카테고리 자동 평가 |
| **투명한 프로세스** | 모든 상태 변경 이력 추적, 워크플로우 로그 완전 기록 |
| **빠른 처리** | 기존 대비 3배 빠른 심사 처리 속도 |
| **간편한 신청** | 동적 폼 빌더 기반, 사업별 맞춤 신청서 |

## 5. API 설계

### 5.1 Base URL
- 개발: `http://localhost:8000/api`
- 운영: `https://{domain}/api`

### 5.2 인증 방식
- **JWT (JSON Web Token)** Bearer Authentication
- 토큰 Payload: `{ sub: user_id, role: "admin" | "user" | "reviewer" }`
- 헤더: `Authorization: Bearer <token>`

### 5.3 API 엔드포인트

#### 인증 (Auth)
| Method | Endpoint | 인증 | 설명 |
|--------|----------|------|------|
| POST | `/api/auth/register` | 공개 | 회원가입 (이메일 중복 검사, 비밀번호 bcrypt 해시) |
| POST | `/api/auth/login` | 공개 | 로그인 → JWT 토큰 반환 |
| GET | `/api/auth/me` | 필요 | 현재 사용자 프로필 조회 |

#### 지원사업 (Programs)
| Method | Endpoint | 인증 | 설명 |
|--------|----------|------|------|
| GET | `/api/programs/` | 공개 | 전체 사업 목록 조회 |
| POST | `/api/programs/` | 관리자 | 신규 사업 생성 |
| GET | `/api/programs/{id}` | 공개 | 사업 상세 조회 |
| PUT | `/api/programs/{id}` | 관리자 | 사업 정보 수정 |
| DELETE | `/api/programs/{id}` | 관리자 | 사업 삭제 |

#### 폼 빌더 (Forms)
| Method | Endpoint | 인증 | 설명 |
|--------|----------|------|------|
| POST | `/api/forms/` | 관리자 | 신청서 폼 생성 (사업 연결) |
| GET | `/api/forms/{id}` | 공개 | 폼 조회 (필드 포함) |
| GET | `/api/forms/by-program/{program_id}` | 공개 | 사업별 폼 조회 |
| POST | `/api/forms/{id}/fields` | 관리자 | 필드 추가 |
| PUT | `/api/forms/{id}/fields/{field_id}` | 관리자 | 필드 수정 |
| DELETE | `/api/forms/{id}/fields/{field_id}` | 관리자 | 필드 삭제 |
| PUT | `/api/forms/{id}/fields/reorder` | 관리자 | 필드 순서 변경 |

#### 신청서 (Applications)
| Method | Endpoint | 인증 | 설명 |
|--------|----------|------|------|
| POST | `/api/applications/` | 사용자 | 신청서 생성 (임시저장) |
| POST | `/api/applications/{id}/submit` | 본인 | 최종 제출 |
| POST | `/api/applications/{id}/files` | 본인 | 첨부파일 업로드 |
| GET | `/api/applications/my` | 사용자 | 내 신청 목록 |
| GET | `/api/applications/admin/all` | 관리자 | 전체 신청 목록 (필터 지원) |
| PUT | `/api/applications/admin/{id}/status` | 관리자 | 상태 변경 + 코멘트 |
| GET | `/api/applications/{id}` | 본인/관리자 | 신청서 상세 조회 |
| GET | `/api/applications/{id}/pdf` | 본인/관리자 | PDF 다운로드 |

#### 대시보드 (Dashboard)
| Method | Endpoint | 인증 | 설명 |
|--------|----------|------|------|
| GET | `/api/dashboard/summary` | 관리자 | 통합 통계 (신청수, 상태별, 선정률, 예산 등) |
| GET | `/api/dashboard/recent-applications` | 관리자 | 최근 신청 목록 (신청자명, 사업명 포함) |

                          └────────┬─────────┘
                                   │ [관리자] 검토 시작
                                   ▼
                          ┌──────────────────┐
                     ┌────┤under_review(검토중)├────┐
                     │    └──────────────────┘    │
                     │                            │
        [관리자] 보완요청                    [관리자] 선정/탈락
                     │                            │
                     ▼                     ┌──────┴──────┐
          ┌────────────────────┐           ▼             ▼
          │revision_requested  │   ┌────────────┐ ┌───────────┐
          │  (보완요청)         │   │ approved   │ │ rejected  │
          └────────┬───────────┘   │  (선정)     │ │  (탈락)    │
                   │               └──────┬─────┘ └───────────┘
                   │ [사용자] 보완 후 재제출     │
                   │                      │ [관리자] 정산 완료
                   ▼                      ▼
          ┌──────────────────┐   ┌──────────────┐
          │ submitted (접수)  │   │ completed    │
          │  ↑ 재접수         │   │  (정산완료)   │
          └──────────────────┘   └──────────────┘
```

### 상태값 정의
| 상태 코드 | 한글명 | 설명 |
|-----------|--------|------|
| `draft` | 임시저장 | 작성 중 (미제출) |
| `submitted` | 접수 | 제출 완료, 접수 확인 대기 |
| `under_review` | 검토중 | 관리자/AI 심사 진행 중 |
| `revision_requested` | 보완요청 | 서류 보완 필요 |
| `approved` | 선정 | 심사 통과, 지원 대상 선정 |
| `rejected` | 탈락 | 심사 미통과 |
| `completed` | 정산완료 | 사업 종료 및 정산 완료 |

## 7. 인증 및 보안

### 7.1 인증 체계
- **JWT (JSON Web Token)** 방식
- 비밀번호: **bcrypt** 해시 (salt 자동 적용)
- 토큰 만료: 설정 가능 (기본 24시간)

### 7.2 권한 체계
| 역할 | 코드 | 접근 범위 |
|------|------|-----------|
| 관리자 | `admin` | 모든 기능 (사업 관리, 폼 빌더, 심사, 대시보드) |
| 심사위원 | `reviewer` | 신청서 열람 및 심사 (향후 확장) |
| 일반 사용자 | `user` | 사업 조회, 신청서 작성/제출, 내 신청 조회 |

### 7.3 라우트 보호
- 프론트엔드: `PrivateRoute` 컴포넌트로 미인증 접근 차단
- 백엔드: `get_current_user()` 의존성 주입으로 API 레벨 인증 검증
- 관리자 라우트: role 체크 추가 적용

## 9. 프론트엔드 구조

### 9.1 라우팅 맵

#### 사용자 포털 (UserLayout)
| 경로 | 컴포넌트 | 인증 | 설명 |
|------|----------|------|------|
| `/` | ProgramList | 공개 | 사업 목록 (랜딩) |
| `/programs/:id` | ProgramDetail | 공개 | 사업 상세 |
| `/login` | UserLogin | 공개 | 사용자 로그인 |
| `/register` | UserRegister | 공개 | 회원가입 |
| `/programs/:id/apply` | ApplyForm | 인증 | 신청서 작성 |
| `/my-applications` | MyApplications | 인증 | 내 신청현황 |

#### 관리자 포털 (AdminLayout)
| 경로 | 컴포넌트 | 인증 | 설명 |
|------|----------|------|------|
| `/admin/login` | AdminLogin | 공개 | 관리자 로그인 |
| `/admin` | Dashboard | 관리자 | 대시보드 |
| `/admin/programs` | Programs | 관리자 | 사업 관리 |
| `/admin/programs/:id/form` | FormBuilder | 관리자 | 폼 빌더 |
| `/admin/applications` | Applications | 관리자 | 신청 현황 |
| `/admin/applications/:id` | ApplicationDetail | 관리자 | 신청서 상세 심사 |

### 9.2 상태 관리
- **Zustand** 기반 `authStore`
  - `token`: JWT 액세스 토큰 (localStorage 영속)
  - `user`: 현재 로그인 사용자 정보
  - `login()`: 로그인 처리
  - `logout()`: 로그아웃 (토큰/사용자 초기화)
  - `fetchUser()`: 토큰으로 사용자 정보 로드

### 9.3 디자인 시스템
- **Pretendard** 폰트 (한글 최적화)
- **Custom Color Palette**: primary(blue), accent(orange), surface(gray), success/warning/danger
- **Premium UI 클래스**: `card-hover`, `btn-primary`, `btn-secondary`, `input`, `badge`
- **반응형 디자인**: 모든 페이지 모바일/태블릿/데스크톱 지원
- **애니메이션**: `animate-fade-in`, `animate-float` 등 커스텀 애니메이션

## 10. 배포 구성

### 10.1 Docker Compose 서비스
| 서비스 | 이미지 | 포트 | 설명 |
|--------|--------|------|------|
| db | postgres:15-alpine | 5432 | PostgreSQL 데이터베이스 |
| minio | minio/minio:latest | 9000, 9001 | S3 호환 오브젝트 스토리지 |
| backend | ./backend (빌드) | 8000 | FastAPI 백엔드 |
| frontend | ./frontend (빌드) | 3000 → 80 | Nginx + React SPA |

### 10.2 환경 변수
| 변수 | 설명 | 예시 |
|------|------|------|
| DATABASE_URL | DB 연결 문자열 | `postgresql://baikal_user:password@db:5432/baikal_grant_db` |
| SECRET_KEY | JWT 서명 키 | (랜덤 생성) |
| S3_ENDPOINT | MinIO 엔드포인트 | `minio:9000` |
| S3_ACCESS_KEY | MinIO 접근 키 | `minioadmin` |
| S3_SECRET_KEY | MinIO 비밀 키 | `minioadmin` |
| S3_BUCKET | 파일 버킷명 | `baikal-files` |
| USE_LOCAL_STORAGE | 로컬 파일 저장 사용 | `true` |

### 10.3 폐쇄망 배포

# 이미지 추출
docker save baikal_backend baikal_frontend postgres:15-alpine minio/minio:latest \
  | gzip > baikal-grant-ai.tar.gz

# 폐쇄망 서버에서 로드
docker load < baikal-grant-ai.tar.gz
docker compose up -d
```
